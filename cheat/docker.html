Start a Container

docker run [flags] {image}

{image} is the image name from Docker Hub*

Extra flags:
  -d, --detach
        Run in detached mode (gives you the console back, rather than logs)
  --name {something}
        Name your container something memorable (like Eric)
  -p {local port}:{remote port}
        Maps remote port on the container to local port at localhost
          eg - -p 8080:80 would mean you could hit http://localhost:8080 and you'd get the same result as hitting http://{docker_ip}:80/
        You can use this flag multiple times to map multiple ports / rangers
  -? --???
        Set environmental variables to
  -v {localVolume}:{remoteVolume}[:{options}]
        Mount a volume to the container, local must be an absolute path (no ./something)
        Options are :ro or :rw, for read-only or read-write, respectively

Investigate a Container

docker ps [options]
Extra flags:
  -a, --all
        Show all containers (default is only running containers)
Output:

See details for a container:

docker inspect [flags] {image}

{image} is the name you gave the container when you started it above

Misc tricks

Sometimes, you just want to jump onto the box (container) and look around...

docker exec -t -i {container name/id} /bin/bash
gives you this option - assuming that the base image used is one that comes with bash (eg ubuntu)
This can also be used for running any other scripts etc on each container, for example, kafka might come with create_topics.sh, which you can run the same way.
